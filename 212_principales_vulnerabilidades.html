2.1.2.- Principales vulnerabilidades

Ahora que ya conocemos las diferentes herramientas que nos proporcionan seguridad, vamos a ver cuáles son las principales vulnerabilidades que podemos encontrarnos en nuestros sitios web.

Clickjacking

Consiste en intentar engañar al usuario creando varias capas en las que el usuario hace clic en otro enlace no seguro pensando que está en su aplicación. Es de los ataques más comunes.

Inyección SQL

Este ataque se basa en el intento de inyectar sentencias SQL en algún campo de entrada de datos para manipular la base de datos.

Ejecución remota de código

Mediante una vulnerabilidad conocida en el sitio web inyecta un malware en el sitio, el cual permite tomar control del sistema que gestiona este sitio mediante la ejecución de código arbitrario.

Cross-site Request Forgery (CSRF)

Se basa en hacer que el usuario haga acciones que comprometan la seguridad de la aplicación web sin que el mismo se percate de que está siendo engañado.

Cross Site Scripting (XSS)

Es una vulnerabilidad la cual le permite a un atacante la posibilidad de ejecutar código malicioso en el navegador de la víctima, pudiendo así secuestrar las cookies de sesión, las credenciales o incluso realizar ataques DDoS entre otros.
